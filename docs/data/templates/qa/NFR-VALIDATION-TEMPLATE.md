# NFR 验证追踪模板
该模板指导 QA 专家或大模型生成模块级 `nfr-tracking.md`，包含 NFR 指标一览、分段说明与行动建议，便于非功能目标持续可视化。

## 模板结构
1. **概览段**：简述当前模块 NFR 验收度（已达标、待复测、风险点），指出关键指标（性能/可靠性/安全/可观测）与优先级故事。
2. **指标表格**：使用 Markdown 表格罗列每类指标、目标、当前状态、跟踪责任人与缓解计划。
3. **风险与缓解措施**：挑出未达标或波动指标，附上具体缓解措施与复测计划。

## 字段说明与对齐
- `指标类别`：可包含性能、可靠性、安全、可观测、可维护等，优先放关键 NFR（来自 `/docs/ARCH.md` 中的 NFR 定义）。
- `目标值`：直接引用 PRD/ARCH 中的 NFR 目标，如响应时间、吞吐量、MTTR、安全等级等。
- `当前测量`：填入最新测试/监控数据，注明测量时间/工具。
- `状态`：使用 `达标/待复测/需缓解/未覆盖` 表示进度；重大问题标记 `需缓解` 并说明原因。
- `验证负责人`：NFR 的 Owning Team（QA/Dev/Ops/Sec），便于跨团队交付。
- `验证手段`：列具体工具（如 k6、JMeter、Trivy、Chaos 工具、Stress test）或验证步骤（观测/演练）。
- `缓解措施`/`预计复测`：与 TASK 中的里程碑/轮次绑定（如 Round2 性能），并注明复测时间。
- `相关 Story/Task`：指向 `/docs/PRD.md` 的 Story、`/docs/TASK.md` 的任务 ID，确保非功能验证可追溯。

## 附加内容（大模型生成指南）
- **关键风险序列**：列出当前未达标指标、原因、预计复测窗口与影响范围，便于 QA 进行协调。
- **自动化验证记录**：可提供链接（如 k6 报告、Alertmanager 记录、安全扫描报告）作为证据。
- **Traceability**：生成后同步 `/docs/data/traceability-matrix.md`，将 Story → 指标 → 验收状态映射写入，方便回归验证。
- **提醒**：在生成 `nfr-tracking.md` 前，先确认最新的 NFR 定义与里程碑（PRD/ARCH/TASK），避免陈旧指标误导决策。

## 使用建议
- 大模型生成此文件时，可先获知模块名、NFR 目标（PRD/ARCH）、当前监控数据与任务里程碑，再根据本模板输出表格与说明段落；QA 最后复核值的准确性与复测计划。
