# CHANGELOG 分卷规则与索引

> 用于记录根目录 `CHANGELOG.md` 拆分后的历史分卷命名、覆盖范围与维护人，确保所有阶段专家在引用历史变更时有统一入口。

## 分卷命名规则
- **优先按季度归档**：`CHANGELOG-YYYYQ{quarter}.md`，例如 `CHANGELOG-2025Q1.md`，覆盖该季度所有条目。
- **迭代驱动项目**：使用 `CHANGELOG-iter-{iteration}.md`，例如 `CHANGELOG-iter-23.md`，对应内部迭代编号。
- 若季度/迭代仍过大，可追加主版本维度（如 `CHANGELOG-v3-2025Q1.md`）；若改回按年份或模块分卷，需在本节说明。
- 新增分卷后需要在根 `CHANGELOG.md` 顶部“历史记录索引”段落添加指向本文件的链接。

## 维护步骤
1. 评估是否达到拆分阈值（参考 `docs/CONVENTIONS.md` 中的“CHANGELOG 拆分规范”）。
2. 新建目标分卷文件（命名遵循上述规则），并在文件开头注明覆盖范围与维护人。
3. 将待归档条目从根 `CHANGELOG.md` 移动到新分卷，保持时间倒序与原始 Markdown 结构。
4. 更新本 README 的“索引表”，登记分卷文件名、时间/版本范围、维护人及备注。
5. 在根 `CHANGELOG.md` 顶部的历史索引中添加该分卷链接，随后通过常规评审流程提交。
